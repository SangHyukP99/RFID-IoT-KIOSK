### 🛠️ 세부 구현 사항

#### 1. 라즈베리 파이 기반 통합 TCP 소켓 서버 및 네트워크 인프라 구축
* **I/O 다중화 & 멀티스레딩**: 라즈베리 파이 환경에서 `select()` 기반의 I/O 다중화와 `pthread` 멀티스레드를 채택하여 STM32 키오스크 및 다수의 아두이노 반납기의 동시 접속을 효율적으로 관리합니다.
* **실시간 병렬 처리**: 서버가 'IDLE' 상태를 유지하다가 클라이언트 요청 시 독립적인 스레드를 생성하여 논블로킹(Non-blocking) 병렬 처리를 수행합니다.

#### 2. MariaDB 연동 및 보안 전용 계정 권한 제어
* **데이터베이스 연동**: C/C++ API 라이브러리를 활용해 RFID 태그에 매핑된 도서 목록 및 대출/반납 이력 테이블을 실시간 연동했습니다.
* **계정 권한 제어**: 독립적인 `iot` 전용 DB 계정을 생성하고 최소 권한만 부여하여 시스템 보안성과 데이터 신뢰성을 확보했습니다.

#### 3. STM32 MCU 기반 대출 키오스크 하드웨어 시스템 (FSM 설계)
* **FSM 기반 상태 제어**: 유휴, 카드 인식, 대출 처리 등 사전에 정의된 FSM(Finite State Machine) 구조에 따라 동작합니다.
* **하드웨어 제어**:
  * **SPI**: RC522 RFID 리더기를 통한 회원증 및 도서 고속 인식
  * **I2C**: CLCD 디스플레이 상태 출력
  * **GPIO**: 부저 및 LED 피드백 회로 제어

#### 4. 아두이노 기반 분산형 무인 반납기 & Wi-Fi 통신
* **독립형 무인 반납기**: Arduino Uno와 ESP8266 Wi-Fi 모듈을 연동하여 원격 분산 배치가 가능하도록 구현했습니다.
* **통신 안정화**: 수신 루프 대기 시간을 200ms 단위로 개별 제어하여 ESP8266 모듈의 데이터 충돌을 방지하고 바이트 단위 패킷 파싱을 수행했습니다.

#### 5. 데이터 패킷 프로토콜 설계 및 연체 관리 MISSION 시스템
* **고유 프로토콜 정의**: 구분자 기호를 활용한 표준 데이터 패킷을 정의하여 데이터 통신 오류를 최소화했습니다.
* **연체 미션 이벤트를 통한 상태 복구**: 
  * 서버가 DB를 조회하여 사용자가 연체자일 경우 특정 미션 이벤트를 발동합니다.
  * 미션 성공 패킷 수신 시, 서버가 DB의 유저 상태를 '정상'으로 자동 복구하는 제어 로직을 구현했습니다.

### 🔄 시스템 데이터 흐름 (System Flow)

> **통합 데이터 흐름 요약**  
> 1. 라즈베리 파이 소켓 서버 가동 (`select` / `pthread` / IDLE 대기)  
> 2. 클라이언트 접속 및 구분자 기반 데이터 패킷 파싱 (회원/도서 인증)  
> 3. Wi-Fi(ESP8266) 및 Bluetooth(HC-05) 이종 무선망을 통해 SQL 커맨드 실시간 송수신  
> 4. 서보모터 제어, FND/LCD 디스플레이 출력 및 무인 반납기 원격 제어 완료
